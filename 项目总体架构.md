# 房屋租赁展示系统 - 全栈应用

这是一个完整的房屋租赁展示系统，包含前端展示页面、后端API服务器和后台管理系统。

## 🏠 系统架构

```

│   前端展示页面   │      │   后端API服务器   │      │  后台管理系统   │
│   (index.html)  │──────│  (Express+SQLite)│──────│   (Vue 3)      │
│                 │      │                 │      │                 │
│ - 全景展示      │      │ - RESTful API   │      │ - 登录认证     │
│ - 图片画廊      │      │ - 数据库管理     │      │ - 预约管理     │
│ - 房屋信息      │      │ - 跨域支持       │      │ - 状态筛选     │
│ - 预约表单      │      │                 │      │ - 详情查看     │
└─────────────────┘      └─────────────────┘      └─────────────────┘
        │                        │                        │
        └────────────────────────┴────────────────────────┘
                           共享数据库 (SQLite)
```

## 🚀 快速开始

### 1. 安装依赖

#### 后端服务器
```bash
cd server
npm install
```

#### 后台管理系统
```bash
cd admin
npm install
```

### 2. 启动服务

#### 启动后端服务器
```bash
cd server
npm start
# 或
./start-server.sh
```

服务器将在 `http://localhost:3000` 启动

#### 启动后台管理系统
```bash
cd admin
npm run dev
# 或
./start-admin.sh
```

管理系统将在 `http://localhost:8080` 启动

#### 访问前端页面
直接访问 `http://localhost:3000` 即可看到前端展示页面

### 3. 使用流程

1. **访问展示页面**: 打开 `http://localhost:3000`
2. **提交预约**: 在展示页面填写预约表单并提交
3. **管理预约**: 打开 `http://localhost:8080` 登录后台管理
4. **查看预约**: 在后台系统中查看、编辑和管理预约

## 🔌 API接口

### 基础URL
```
http://localhost:3000/api
```

### 主要接口

#### 预约管理
- `GET /appointments` - 获取预约列表（支持分页、筛选）
- `GET /appointments/:id` - 获取单个预约详情
- `POST /appointments` - 创建新预约
- `PUT /appointments/:id` - 更新预约状态
- `PUT /appointments/:id/details` - 更新预约详细信息
- `DELETE /appointments/:id` - 删除预约

详细API文档请查看 [server/README.md](server/README.md)

## 💾 数据库结构

### appointments 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| name | TEXT | 姓名（必填） |
| phone | TEXT | 联系电话（必填） |
| email | TEXT | 电子邮箱（可选） |
| appointment_date | TEXT | 预约日期（必填） |
| message | TEXT | 留言（可选） |
| status | TEXT | 状态 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

## 🔧 配置说明

### 后端服务器配置
在 `server/.env` 文件中配置：

```env
PORT=3000
NODE_ENV=development
```

### 后台管理系统配置
在 `admin/vite.config.js` 中配置代理：

```javascript
proxy: {
  '/api': {
    target: 'http://localhost:3000',
    changeOrigin: true
  }
}
```

## 🐛 故障排查

### 常见问题

#### 1. 后端服务器无法启动
- 检查是否安装了所有依赖：`npm install`
- 检查端口是否被占用
- 查看错误日志

#### 2. 表单提交失败
- 确认后端服务器正在运行
- 检查网络请求是否被阻止
- 查看浏览器控制台错误信息

#### 3. 后台管理系统无法访问
- 确认后台服务器正在运行
- 检查代理配置是否正确
- 确认依赖已安装

#### 4. 数据库文件找不到
- 数据库文件会自动创建在 `server/data/` 目录
- 确认目录权限正确
