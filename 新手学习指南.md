### 🔑 关键概念解释

#### 1. 什么是后端服务器？

**通俗理解**: 就像一个餐厅的后厨
- 前端（展示页面）= 餐厅大厅，客人点菜的地方
- 后端服务器 = 后厨，负责处理订单、做菜（处理数据）
- 数据库 = 仓库，存储食材（存储数据）

**技术理解**: 运行在服务器上的程序
- 接收前端的请求（比如"我要提交预约表单"）
- 处理业务逻辑（比如验证数据是否合法）
- 操作数据库（比如把数据保存起来）
- 返回结果给前端（比如"提交成功"）

#### 2. 什么是API？

**通俗理解**: 前端和后端之间的"菜单"
- API定义了前端可以点的"菜"（可以调用的功能）
- 比如："创建预约"、"查询预约列表"、"修改预约状态"

**技术理解**: 接口定义
- URL地址：`/api/appointments`
- 方法：GET（查询）、POST（创建）、PUT（修改）、DELETE（删除）
- 参数：前端需要传什么数据
- 返回：后端会返回什么数据

#### 3. 什么是数据库？

**通俗理解**: 一个超级Excel表格
- 存储结构化数据
- 可以快速查找、修改、删除数据
- 支持多用户同时访问

**技术理解**: 数据存储系统
- SQLite：轻量级数据库，一个文件就是一个数据库
- 表：存储一类数据（比如appointments表存储预约信息）
- 字段：数据的各个属性（比如姓名、电话、日期）

## 3. 后端开发入门

### 📂 后端文件结构详解

#### server/server.js - 主程序入口

这是整个后端的核心文件，让我们逐行理解：

```javascript
// 1️⃣ 引入依赖包（就像前端引入库）
const express = require('express');     // Express框架（类似Vue框架）
const cors = require('cors');           // 跨域处理（允许前端访问）
const path = require('path');           // 路径处理（处理文件路径）
require('dotenv').config();             // 环境变量（读取.env文件）

// 2️⃣ 创建Express应用（就像创建Vue应用）
const app = express();
const PORT = process.env.PORT || 3000;  // 设置端口

// 3️⃣ 配置中间件（处理请求前的准备工作）
app.use(cors());                         // 允许跨域
app.use(express.json());                 // 解析JSON数据
app.use(express.urlencoded({ extended: true })); // 解析表单数据

// 4️⃣ 提供静态文件服务（让前端页面能被访问）
app.use(express.static(path.join(__dirname, '..')));
// 这行代码让 server/../index.html 可以被访问

// 5️⃣ 设置API路由（定义接口）
app.use('/api/appointments', appointmentsRouter);
// 访问 /api/appointments 就会使用 appointmentsRouter 处理

// 6️⃣ 启动服务器
app.listen(PORT, () => {
    console.log(`服务器运行在 http://localhost:${PORT}`);
});
```

**关键理解**：
- Express = Node.js的Web框架（就像前端有Vue、React）
- 中间件 = 处理请求前的准备工作（比如解析数据、处理跨域）
- 路由 = 定义不同的API接口（不同的URL对应不同的处理逻辑）

#### server/routes/appointments.js - API接口实现

这个文件定义了预约管理的所有API：

```javascript
// 📌 获取预约列表
router.get('/', (req, res) => {
    // req = request（请求信息）
    // res = response（响应信息）
    
    // 1. 从数据库查询数据
    const appointments = db.prepare('SELECT * FROM appointments').all();
    
    // 2. 返回JSON格式数据给前端
    res.json({
        success: true,
        data: appointments
    });
});

// 📌 创建新预约
router.post('/', (req, res) => {
    // req.body = 前端传来的数据
    const { name, phone, email, appointment_date, message } = req.body;
    
    // 1. 验证数据是否合法
    if (!name || !phone || !appointment_date) {
        return res.status(400).json({
            success: false,
            message: '必填项不能为空'
        });
    }
    
    // 2. 插入数据库
    const stmt = db.prepare(`
        INSERT INTO appointments (name, phone, email, appointment_date, message)
        VALUES (?, ?, ?, ?, ?)
    `);
    stmt.run(name, phone, email, appointment_date, message);
    
    // 3. 返回成功消息
    res.json({
        success: true,
        message: '预约成功'
    });
});
```

**关键理解**：
- `req.body` = 前端发送的数据（就像Vue中的props）
- `res.json()` = 返回JSON数据给前端（就像API响应）
- `router.get()` = 定义GET接口（查询数据）
- `router.post()` = 定义POST接口（创建数据）

## 4. 数据库基础知识

### 🗄️ SQLite数据库介绍

#### 为什么选择SQLite？

**优点**（适合初学者）：
- ✅ 不需要安装MySQL等大型数据库软件
- ✅ 一个文件就是一个数据库
- ✅ 轻量级，适合小型项目
- ✅ 学习成本低，语法简单

**缺点**（不适合大型项目）：
- ❌ 不适合高并发（多人同时访问）
- ❌ 不支持复杂的数据库操作

#### 数据库操作基础

**SQL语法**（就像Excel的公式）：

```sql
-- 📌 创建表（定义数据结构）
CREATE TABLE appointments (
    id INTEGER PRIMARY KEY AUTOINCREMENT,  -- ID，自动增长
    name TEXT NOT NULL,                     -- 姓名，必填
    phone TEXT NOT NULL,                    -- 电话，必填
    email TEXT,                             -- 邮箱，可选
    appointment_date TEXT NOT NULL,         -- 预约日期，必填
    message TEXT,                           -- 留言，可选
    status TEXT DEFAULT 'pending',          -- 状态，默认待确认
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP  -- 创建时间
);

-- 📌 查询数据（SELECT）
SELECT * FROM appointments;                 -- 查询所有预约
SELECT * FROM appointments WHERE status='pending';  -- 查询待确认的预约

-- 📌 插入数据（INSERT）
INSERT INTO appointments (name, phone, appointment_date)
VALUES ('张三', '13800138000', '2024-12-25');

-- 📌 更新数据（UPDATE）
UPDATE appointments SET status='confirmed' WHERE id=1;

-- 📌 删除数据（DELETE）
DELETE FROM appointments WHERE id=1;
```

**在Node.js中使用SQLite**：

```javascript
// 1️⃣ 引入数据库模块
const Database = require('better-sqlite3');
const db = new Database('appointments.db');

// 2️⃣ 查询数据
const appointments = db.prepare('SELECT * FROM appointments').all();

// 3️⃣ 插入数据
const stmt = db.prepare('INSERT INTO appointments VALUES (?, ?, ?)');
stmt.run('张三', '13800138000', '2024-12-25');

// 4️⃣ 更新数据
db.prepare('UPDATE appointments SET status=? WHERE id=?').run('confirmed', 1);

// 5️⃣ 删除数据
db.prepare('DELETE FROM appointments WHERE id=?').run(1);
```

## 5. 前端到后端的通信

### 🌐 HTTP请求详解

#### 前端如何发送数据到后端？

让我们看前端代码：

```javascript
// 📌 script.js 中的表单提交代码
async function submitForm() {
    // 1️⃣ 收集表单数据
    const formData = new FormData(form);
    const data = {
        name: formData.get('name'),
        phone: formData.get('phone'),
        email: formData.get('email'),
        appointment_date: formData.get('date'),
        message: formData.get('message')
    };
    
    // 2️⃣ 发送HTTP请求
    const response = await fetch('http://localhost:3000/api/appointments', {
        method: 'POST',                    // POST方法（创建数据）
        headers: {
            'Content-Type': 'application/json'  // 告诉后端这是JSON数据
        },
        body: JSON.stringify(data)         // 把JavaScript对象转成JSON字符串
    });
    
    // 3️⃣ 接收后端返回的数据
    const result = await response.json();  // 解析JSON响应
    
    // 4️⃣ 处理结果
    if (result.success) {
        alert('提交成功！');
    } else {
        alert('提交失败：' + result.message);
    }
}
```

#### HTTP请求方法对比

| 方法 | 用途 | 举例 |
|------|------|------|
| GET | 查询数据 | 查询预约列表 |
| POST | 创建数据 | 提交新预约 |
| PUT | 更新数据 | 修改预约状态 |
| DELETE | 删除数据 | 删除预约 |

#### 数据格式：JSON

**什么是JSON？**
- JavaScript Object Notation（JavaScript对象表示法）
- 前端和后端通信的标准数据格式

**举例**：
```json
{
    "name": "张三",
    "phone": "13800138000",
    "email": "test@example.com",
    "appointment_date": "2024-12-25",
    "message": "希望下午看房"
}
```

**在前端**：
```javascript
// JavaScript对象 → JSON字符串
const jsonString = JSON.stringify(data);

// JSON字符串 → JavaScript对象
const jsObject = JSON.parse(jsonString);
```

## 6. 后台管理系统原理

### 🖥️ Vue项目结构详解

#### admin/src/views/Appointments.vue 核心逻辑

```javascript
// 📌 加载预约数据
const loadAppointments = async () => {
    // 1️⃣ 发送GET请求到后端
    const response = await axios.get('/api/appointments', {
        params: {
            page: pagination.page,

        }

    // 2️⃣ 接收数据并显示
    appointments.value = response.data.data;
    pagination.total = response.data.pagination.total;

```

**关键理解**：
- Vue组件 = 一个页面或页面的一部分
- axios = 专门用于HTTP请求的库（比fetch更简单）
- 数据绑定 = Vue自动把数据显示在页面上

#### 路由系统（Router）

**什么是路由？**
- URL和页面的对应关系
- 比如：`/login` → Login.vue页面
- 比如：`/appointments` → Appointments.vue页面

**配置代码**：
```javascript
const routes = [
    {
        path: '/login',
        name: 'Login',
        component: Login
    },
    {
        path: '/appointments',
        name: 'Appointments',
        component: Appointments
    }
];
```

#### Vite配置（重要）

**vite.config.js - API代理配置**：

```javascript
export default defineConfig({
    server: {
        port: 8080,  // Vue开发服务器端口
        proxy: {
            '/api': {
                target: 'http://localhost:3000',  // 后端服务器地址
                changeOrigin: true
            }
        }
    }
});
```

**为什么需要代理？**
- Vue运行在 `http://localhost:8080`
- 后端运行在 `http://localhost:3000`
- 直接访问会有跨域问题
- 代理让 `/api` 请求自动转发到后端

```bash
cd house-rental-demo/server

cd ../admin
npm install
```

**步骤5: 使用PM2启动后端**

```bash

sudo npm install -g pm2

# 启动服务器
pm2 start server/server.js --name house-rental-api

# 查看状态
pm2 status

# 查看日志
pm2 logs house-rental-api

# 设置开机自启
pm2 startup
pm2 save
```

**步骤6: 配置Nginx**

```nginx
# 安装Nginx
sudo yum install nginx

# 配置文件：/etc/nginx/conf.d/house-rental.conf
server {
    listen 80;
    server_name your-domain.com;  # 你的域名
    
    # 提供前端展示页面
    root /path/to/house-rental-demo;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # API代理
    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
    
    # 后台管理系统
    location /admin {
        alias /path/to/house-rental-demo/admin/dist;
        try_files $uri $uri/ /index.html;
    }
}
```

```bash
# 启动Nginx
sudo systemctl start nginx
sudo systemctl enable nginx  # 设置开机自启

# 重启Nginx（修改配置后）
sudo systemctl restart nginx
```

**步骤7: 访问测试**

```bash

http://your-domain.com/

# 后台管理系统
```

## 10. 常见问题解答

### ❓ Q1: 为什么需要后端服务器？

**A**: 
- 前端只能展示页面，不能存储数据
- 后端可以处理业务逻辑（比如验证手机号格式）
- 后端可以操作数据库（永久保存数据）
- 后端可以提供API接口（前端通过API获取数据）

### ❓ Q2: Express和Vue有什么区别？

**A**:
- **Express**: 后端框架，运行在Node.js上，处理服务器逻辑
- **Vue**: 前端框架，运行在浏览器中，处理页面展示和交互
- 一个项目通常需要前端+后端配合工作

### ❓ Q3: SQLite和MySQL有什么区别？

**A**:
| 特性 | SQLite | MySQL |
|------|--------|-------|
| 安装 | 不需要安装 | 需要安装服务器软件 |
| 适用场景 | 小型项目、学习 | 大型项目、高并发 |
| 数据存储 | 单个文件 | 需要独立服务器 |
| 性能 | 较低 | 较高 |

本项目使用SQLite适合学习和小型应用。

### ❓ Q4: 什么是跨域问题？

**A**:
- 浏览器的安全机制，阻止不同域名的网页访问数据
- 比如：`localhost:8080` 的Vue页面访问 `localhost:3000` 的API
- 解决方案：
  - 后端配置cors中间件
  - 前端配置代理（Vite proxy）

### ❓ Q5: 如何查看数据库中的数据？

**A**:
方法1: 使用SQLiteBrowser工具（推荐）
- 下载：https://sqlitebrowser.org/
- 打开 `server/data/appointments.db` 文件
- 可以查看、修改、删除数据

方法2: 使用命令行
```bash
sqlite3 server/data/appointments.db
sqlite> SELECT * FROM appointments;
```

方法3: 通过API查询
```bash
curl http://localhost:3000/api/appointments
```

### ❓ Q6: 如何添加新的功能？

**A**: 基本步骤：
1. 修改前端表单（添加新字段）
2. 修改前端JavaScript（收集新字段）
3. 修改数据库表结构（添加新字段）
4. 修改后端API（处理新字段）
5. 测试新功能

具体操作参考本文第8节。

### ❓ Q7: 项目可以部署到哪里？

**A**: 推荐平台：
- **云服务器**: 阿里云、腾讯云、AWS
- **免费平台**: 
  - Vercel（支持Node.js）
  - Railway（支持数据库）
  - Heroku（有免费额度）

### ❓ Q8: 如何让其他人访问我的项目？

**A**: 步骤：
1. 购买云服务器或使用免费平台
2. 上传代码到服务器
3. 安装Node.js环境
4. 启动后端服务器（使用PM2）
5. 配置Nginx反向代理
6. 告诉其他人你的域名或IP地址

详细步骤参考本文第7节。
