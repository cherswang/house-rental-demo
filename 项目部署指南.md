## 🎯 方案1: 免费平台部署（推荐新手）

### 优点
- ✅ 完全免费
- ✅ 自动部署，无需手动配置服务器
- ✅ 提供免费域名
- ✅ 支持Node.js和数据库

### 推荐平台组合

#### 组合1: Vercel（前端）+ Railway（后端）

**Vercel**: 部署前端展示页面
**Railway**: 部署后端API和数据库

#### 组合2: Netlify（前端）+ Render（后端）

**Netlify**: 部署前端展示页面  
**Render**: 部署后端API和数据库

## 📦 方案1详细步骤

### 步骤1: 准备代码仓库

#### 1.1 创建GitHub仓库

```bash
# 在项目根目录初始化Git
cd house-rental-demo
git init

# 创建.gitignore文件（排除不需要上传的文件）
cat > .gitignore << 'EOF'
node_modules/
.env
*.log
.DS_Store
data/*.db
dist/
EOF

# 提交代码
git add .
git commit -m "Initial commit"

# 连接GitHub远程仓库（需要先在GitHub创建仓库）
git remote add origin https://github.com/你的用户名/house-rental-demo.git
git push -u origin main
```

#### 1.2 修改项目配置

**修改server/.env**:
```env
# 不要使用localhost，改成0.0.0.0允许外部访问
PORT=3000
NODE_ENV=production
```

**修改前端API地址**:
编辑 `script.js`:
```javascript
// 修改API地址（使用环境变量或相对路径）
const API_BASE_URL = window.location.origin;
```

### 步骤2: 部署后端到Railway（推荐）

#### 2.1 注册Railway

访问: https://railway.app/
- 使用GitHub账号登录
- 无需付费，有免费额度

#### 2.2 创建项目

1. 点击"New Project"
2. 选择"Deploy from GitHub repo"
3. 选择你的仓库
4. 设置部署配置

#### 2.3 配置部署参数

在Railway项目设置中:

**Root Directory**: `server`  
**Build Command**: `npm install`  
**Start Command**: `node server.js`  

#### 2.4 添加环境变量

在Railway的Variables中添加:
```
PORT=3000
NODE_ENV=production
```

#### 2.5 添加SQLite数据库

Railway支持SQLite:
- 数据库文件会自动创建在 `server/data/` 目录
- Railway会持久化存储这个目录

#### 2.6 获取API地址

部署完成后，Railway会提供一个域名:
```
https://your-project.up.railway.app
```

测试API:
```bash
curl https://your-project.up.railway.app/api/appointments
```

### 步骤3: 部署前端到Vercel

#### 3.1 注册Vercel

访问: https://vercel.com/
- 使用GitHub账号登录
- 完全免费

#### 3.2 创建项目

1. 点击"New Project"
2. 选择你的GitHub仓库
3. 配置部署

#### 3.3 配置部署参数

**Framework Preset**: Other  
**Root Directory**: `.` (项目根目录)  
**Build Command**: 不需要构建  
**Output Directory**: `.`  

#### 3.4 配置API代理

Vercel需要配置重定向规则。

创建 `vercel.json`:
```json
{
  "rewrites": [
    {
      "source": "/api/(.*)",
      "destination": "https://your-railway-app.up.railway.app/api/$1"
    }
  ]
}
```

#### 3.5 获取前端地址

部署完成后，Vercel会提供一个域名:
```
https://your-project.vercel.app
```

访问这个地址，就可以看到你的房屋租赁展示页面了！

### 步骤4: 部署Vue后台管理

#### 4.1 构建Vue项目

```bash
cd admin
npm install
npm run build
```

这会生成 `admin/dist` 目录。

#### 4.2 部署到Vercel（子项目）

可以创建一个新的Vercel项目专门部署后台：

1. 创建新的Vercel项目
2. Root Directory设置为 `admin`
3. Build Command: `npm run build`
4. Output Directory: `dist`

#### 4.3 配置API代理

在 `admin/vercel.json`:
```json
{
  "rewrites": [
    {
      "source": "/api/(.*)",
      "destination": "https://your-railway-app.up.railway.app/api/$1"
    }
  ]

```

#### 4.4 访问后台管理系统

```

```

### 步骤1: 购买和配置云服务器

#### 1.1 购买服务器（以阿里云为例）

1. 访问阿里云官网: https://www.aliyun.com/

#### 1.2 连接服务器

**Ubuntu系统**:

#### 1.4 安装PM2进程管理器

### 步骤5: 配置Nginx反向代理

#### 5.1 创建Nginx配置文件

```bash
sudo cat > /etc/nginx/conf.d/house-rental.conf << 'EOF'
server {
    listen 80;
    server_name 你的域名或IP;

    # 前端展示页面
    root /var/www/house-rental-demo;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # API接口代理
    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_cache_bypass $http_upgrade;
    }

    # Vue后台管理系统
    location /admin {
        alias /var/www/house-rental-demo/admin/dist;
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }
}
EOF
```

#### 5.2 测试配置

```bash

# 重启Nginx
sudo systemctl restart nginx
```

```nginx
    listen 80;
    server_name your-domain.com;
```

## 🎯 方案3: Docker容器化部署

### 优点
- ✅ 环境一致性，避免配置问题
- ✅ 快速部署和扩展
- ✅ 适合团队协作

### 步骤1: 创建Dockerfile

#### 1.1 后端Dockerfile

创建 `server/Dockerfile`:
```dockerfile
FROM node:16-alpine

WORKDIR /app

# 复制依赖文件
COPY package*.json ./
RUN npm install --production

# 复制源代码
COPY . .

# 创建数据目录
RUN mkdir -p data

# 暴露端口
EXPOSE 3000

# 设置环境变量
ENV NODE_ENV=production
ENV PORT=3000

# 启动服务
CMD ["node", "server.js"]
```

#### 1.2 前端Dockerfile

创建 `admin/Dockerfile`:
```dockerfile
FROM node:16-alpine as builder

WORKDIR /app

# 复制依赖文件
COPY package*.json ./
RUN npm install

# 复制源代码
COPY . .

# 构建
RUN npm run build

# 使用Nginx提供静态文件
FROM nginx:alpine

# 复制构建结果
COPY --from=builder /app/dist /usr/share/nginx/html

# 复制Nginx配置
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

创建 `admin/nginx.conf`:
```nginx
server {
    listen 80;
    
    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;
    }
    
    location /api {
        proxy_pass http://backend:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 步骤2: 使用Docker Compose

创建 `docker-compose.yml`:
```yaml
version: '3.8'

services:
  backend:
    build: ./server
    container_name: house-rental-api
    restart: always
    ports:
      - "3000:3000"
    volumes:
      - ./server/data:/app/data
    environment:
      - NODE_ENV=production
      - PORT=3000

  frontend:
    build: ./admin
    container_name: house-rental-admin
    restart: always
    ports:
      - "8080:80"
    depends_on:
      - backend
```

### 步骤3: 部署Docker容器

```bash
# 构建并启动所有容器
docker-compose up -d

# 查看容器状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 停止容器
docker-compose down
```

## 🚨 常见问题排查

### 问题1: 网站无法访问

**排查步骤**:
```bash
# 1. 检查Nginx状态
sudo systemctl status nginx

# 2. 检查端口监听
sudo netstat -tlnp | grep :80

# 3. 检查防火墙
sudo firewall-cmd --list-all  # CentOS
sudo ufw status               # Ubuntu

# 4. 查看Nginx错误日志
sudo tail -f /var/log/nginx/error.log
```

### 问题2: API接口500错误

**排查步骤**:
```bash
# 1. 检查PM2进程状态
pm2 status

# 2. 查看后端日志
pm2 logs house-rental-api

# 3. 测试本地API
curl http://localhost:3000/api/appointments

# 4. 检查数据库文件
ls -la server/data/
```

### 问题3: 表单提交失败

**排查步骤**:
```bash
# 1. 查看浏览器Network请求
# 2. 检查API地址是否正确
# 3. 检查CORS配置
# 4. 查看后端错误日志
pm2 logs house-rental-api
```

### 问题4: 后台管理系统404

**排查步骤**:
```bash
# 1. 检查dist目录是否存在
ls -la admin/dist/

# 2. 检查Nginx配置
sudo nginx -t

# 3. 查看Nginx配置文件
cat /etc/nginx/conf.d/house-rental.conf
```

```bash
# 1. 检查数据库文件权限
```

## 📈 性能优化建议

### 1. 启用Gzip压缩

在Nginx配置中添加:
```nginx
gzip on;
gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
gzip_min_length 1000;
```

### 2. 设置静态资源缓存

```nginx
location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
    expires 7d;
    add_header Cache-Control "public, immutable";
}
```

### 3. 优化PM2配置

```javascript
module.exports = {
  apps: [{
    name: 'house-rental-api',
    script: 'server.js',
    instances: 'max',  // 使用所有CPU核心
    exec_mode: 'cluster',  // 集群模式
    max_memory_restart: '1G',
    env: {
      NODE_ENV: 'production',
      PORT: 3000
    }
  }]
};
```
